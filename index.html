¿En qué consiste el proyecto?

El objetivo principal de este proyecto es documentar los pasos seguidos en la elaboración de un brazo robótico a costo accesible. Algunos de los softwares utilizados se enlistan a continuación:

Matlab: AppDesigner

Permite realizar aplicaciones que funcionan en paralelo con scripts y programas, tanto de Matlab como de Simulink,se realizó una aplicación que permite conectarse con el robot.

Robotics System Toolbox

Como primer paso se generó el modelado en 3D del brazo robótico controlable de forma dinámica por Simscape y Robotics System Toolbox utilizando el modelo de SolidWorks de este.

Simulink

Se compiló un programa de Matlab-Simulink por medio de un script que se encarga de generar las variables para poder ser utilizado dentro de un bloque de control cinemático.

Al terminar el proyecto

Luego de trabajar durante todo el semestre se obtuvieron los siguientes resultados, conclusiones y recomendaciones.

Resultados

Debido a la complejidad del proyecto, fue necesario hacer distintas iteraciones en el modelado 3D del robot. El cambio más importante fue dejar de lado los motores NEMA 43, debido a las restricciones económicas que representaba maquinar piezas que lograran sostener el peso de estos motores. Se optó por usar dos motores NEMA 23, los cuales se ubican en la parte inferior del robot y son usados para crear el movimiento esférico de la base dando movimiento al primer y segundo eje. Adicionalmente, se diseñó una tarjeta personalizada para el control de los motores en el software easyEDA, la cual se muestra en la imagen 21, que opera con un microcontrolador ATmega32U4 correspondiente al arduino Leonardo usado para las pruebas

Conclusiones

A nivel mecánico fue necesario modificar los planes inciales y optar por recursos alternos con tal de satisfacer los objetivos proyectados, sin embargo, a pesar de las dificultades y los contratiempos se logró diseñar, maquinar y ensamblar el modelo que fue considerado óptimo.

Consideraciones y recomendaciones

Recomendaciones incluyen incrementar la relación de los reductores, agregar dos reductores más en los ejes 4 y 5, e implementar un sistema de frenos NO (Normalmente Cerrados) en todos los ejes del brazo robótico, esto último permitiría que en caso de un corte energético las posiciones de los ejes se mantengan. Adicionalmente, se recomienda que el software tenga la posibilidad de crear y guardar rutinas complejas para enriquecer las funciones del robot mediante la interfaz de usuario.
